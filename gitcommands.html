Top 9 Most Used Git Commands For Developers

1. git config

Before start using Git, it needs to be configured properly. This command allows us to specify the username and email address that will be used for our commits.

  • # sets up Git with your name

  • git config --global user.name "{Full-Name}"

  • # sets up Git with your email

  • git config --global user.email "{email-address}"

2. git clone

The git clone command takes a url to the Git repository we want to clone. The path to the Git repository you want to clone is passed as an argument to the git clone command. git clone command produces a local working copy of a remote repository's source code. When we clone a repository, the code is downloaded to our machine automatically.

$ git clone https://github.com/{repo-url}

3. git branch

Add a new branch to an existing repository, list all existing branches, and delete a branch with git branch. This command is used to perform operations on a specified branch. Here is how we create a new branch locally:

$ git branch {branch-name}

Take a look at the branches and check out on which branch we are currently working:

$ git branch or $ git branch --list

4. git checkout

git checkout command is used to switch to an existing branch or to create and switch to a new one. The branch you want to switch to must be present in our local machine, and any modifications made to your current branch must be committed or stashed before switching.

$ git checkout {branch-name}

5. git pull

We can fetch and integrate the content of the remote repository into our local repository using this command. git pull pulls the most recent changes from the remote server into the local repository, ensuring you have the most up-to-date information from your coworkers.

$ git pull

6. git add

git add command is used to move files from the Working Directory to the Staging area. The git add command saves our changes in a file to the staging area, allowing you to compare your local version to the version of remote repository. We use git add command to add a file to the staging area before committing our new or changed file. To add specific files:

$ git add {file1} {file2} â€¦ {fileN}

To add all the files:

$ git add .

7. git commit

This command saves a log message along with the commit id of the changes made to the git repository. The modifications are saved in our local repository with git commit. We must include a brief description of the changes made every time you commit our code changes for better understanding.

$ git commit -m "{your commit message here}"{repo-url}

8. git push

This command pushes the contents of our local repository to the remote repository.

$ git push{repo-url}

9. git merge

Using this command we can merge the new chages in the specified branch into the current branch. The command git merge joins your branch to the parent branch.If there are no conflicts, it will make a new commit automatically.

$ git merge {name-of-branch-to-merge-in}
